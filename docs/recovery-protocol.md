# Recovery Protocol

## Goal

Enable a new machine, new agent, or new collaborator to recover the active memory context with minimal ambiguity.

## Recommended Read Order

1. `README.md`
2. `SYSTEM.md`
3. `INDEX.md`
4. Organization-level overview and policies
5. Team-level overview and project index
6. Relevant project status
7. Relevant resource mappings
8. Relevant people and agent files

## Recovery Outcome

A recovery is considered successful when the reader can answer:

- What system is this?
- What scope am I operating in?
- What projects are active?
- What resources are involved?
- What rules and constraints apply?
- What should happen next?

## Recovery Rules

- Read broader scope before narrower scope
- Read status before tasks when resuming work
- Use mapping files to identify dependencies
- Do not infer hidden permissions from repository structure alone
